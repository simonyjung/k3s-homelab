# Notes

Operations journal for the cluster — what's being worked on and what got done.
For documentation of *how the cluster works*, see [`docs/`](../docs/).

- [`plan.md`](./plan.md) — active improvement items and their status.
- [`history/`](./history/) — completed items, archived one file per quarter (`YYYY-qN.md`).

Convention: a PR that completes a plan item also updates `plan.md` in the same PR,
moving the finished item to the current quarter's history file. When a new quarter
starts, the next completed item simply opens a new `history/YYYY-qN.md`.
